### Engineering view (reader/writer symmetry)

`ROTATED_RE = /^app-\d{4}-\d{2}-\d{2}(?:\.\d+)?\.log(?:\.gz)?$/`. Export
it and reuse it in `src/logs/reader.js` rather than duplicating. One
source of truth.

### Security / safety view
- The new gunzip path reads from a server-controlled directory; no
  user-supplied filenames. Still: pin the decompressed-size cap (the
  zip-bomb guard) at `config.maxLogReadMb` and abort the stream early if
  exceeded.
- Don't introduce a dependency. Use only `node:zlib` and
  `node:stream/promises`.

### Documentation view

per `~/.claude/rules/common/release-docs.md`. The 15→16 fix is a one-line
correction; cross-check there are no other "15" stragglers.

### Risk view
The reader change is the only one with non-trivial risk (streaming I/O,
binary input). Mitigation: write the new tests first (TDD per
`~/.claude/rules/common/testing.md`), then implement. A gzip fixture is
two lines: `zlib.gzipSync(Buffer.from(ndjson))`.
