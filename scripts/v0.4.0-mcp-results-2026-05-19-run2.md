**Issue flagged (3.1–3.4 + 4.3):** Guardrails redact mode injects a
`_guardrails_banner` field into the forwarded JSON body. Mistral's chat-completions
schema is `extra_forbidden` and rejects the request with 422. The redaction itself
works correctly (raw secret bytes never leave the proxy) but the upstream
round-trip fails. Options: (a) inject the banner as a header instead of a body
field; (b) place the banner inside an assistant/system message; (c) document
"redact mode is incompatible with strict-schema upstreams". This regressed since
the previous v0.4.0 run on the same branch where 3.1–3.4 returned 200.
